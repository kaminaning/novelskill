## 第二部分:不可算指标(需正文样本)

> 以下指标**无法从榜单元数据推出**,必须有实际章节正文才能算。
> 当 write skill 的 Phase 0 走模式 B/A 时,会用以下算法实时计算。

### 段落级指标(必须正文样本)

| 指标 | 算法 | 番茄风目标值 |
|------|------|-------------|
| 平均段长 | 总字符数 / 总段数 | 30-80 字 |
| 段长中位数 | 段长排序后的中位 | 35-60 字 |
| 单句独立段占比 | 单句段数 / 总段数 × 100% | ≤30% |
| 连续 ≥3 段 <10 字短段堆叠 | 滑动窗口扫描 | 每章 0 次 |
| 对话段占比 | 含引号"…"或「…」的段数 / 总段数 | 30-50%(对话推动) |
| 视觉爆点独行段 | <10 字且无引号的独立段 | 每章 ≤5 处 |
| 心理戏总量 | 含"想/觉得/心里/暗道"等词的段字数 | ≤100 字/章 |

### 计算样本要求

- **最少**:3 章正文(用户给文件路径)
- **推荐**:5-10 章正文(覆盖开篇/中段/章末三种节奏)
- **理想**:整本书前 30 章(覆盖死亡开局 + 副本切换)

### 算法实现(供 write skill Phase 0 模式 B 参考)

```python
import re

def analyze_paragraph_style(text):
    """输入一章正文,返回文风指标字典"""
    paras = [p.strip() for p in text.split("\n\n") if p.strip()]
    total = len(paras)
    if total == 0: return None

    para_lens = [len(p) for p in paras]
    single_sentence_paras = sum(1 for p in paras if p.count("。") + p.count("！") + p.count("？") <= 1)
    dialogue_paras = sum(1 for p in paras if '"' in p or '"' in p or "「" in p)
    visual_burst = sum(1 for p in paras if len(p) < 10 and not any(c in p for c in '"「'))

    # 连续短段堆叠
    consec_short = 0
    max_streak = 0
    cur_streak = 0
    for p in paras:
        if len(p) < 10:
            cur_streak += 1
            max_streak = max(max_streak, cur_streak)
            if cur_streak >= 3:
                consec_short += 1
        else:
            cur_streak = 0

    return {
        "段数": total,
        "平均段长": sum(para_lens) // total,
        "段长中位数": sorted(para_lens)[total//2],
        "单句段占比": round(single_sentence_paras / total * 100, 1),
        "对话段占比": round(dialogue_paras / total * 100, 1),
        "视觉爆点独行": visual_burst,
        "连续短段堆叠次数": consec_short,
        "最长短段连续": max_streak,
    }
```

## 最后提醒

### 兜底 ≠ 万能

本文件只是兜底,实际用 analyze-data 的文风分析(模式 A)会**精准得多**,因为:
- 模式 A 算的是单本书的真实段落结构
- 模式 C 是市场平均(不针对你的对标书)

**强烈建议**:开新书前花 30 分钟用 `/story-long-analyze` 把对标书拆掉,后续写作期 0 心智负担。

### 数据 ≠ 模板

本文件给的是"番茄市场长什么样",不是"你必须写成什么样":
- 章长目标是参考区间,不是硬性指标
- 单句段占比上限 30% 是底线,不是上限
- 章名样式占比反映"什么有效",不是"必须用这个比例"

**核心原则**:数据约束底线,创作决定上限。
